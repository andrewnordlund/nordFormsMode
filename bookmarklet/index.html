Forms Mode Simulator Bookmarklet

What it does

Some screen readers (like JAWS and NVDA) have something called "Forms Mode" or "Focus Mode" where keystrokes are sent directly to the browser instead of the sreen reader for interpreation. In normal mode or "Browse Mode" or "Document Mode", for example, pressing "H" takes you to the next <h#>. But in Forms Mode, like if you're filling in a form, then pressing "H" causes an "H" to go into the input you're focused on. In Forms Mode, to navigate, users can either exit Forms Mode and go to the next thing (but you can't guarantee every user will do this), or the user can hit Tab to move to the next focusable element. In Forms Mode, it's a good practice to assume that elements that are not focusable nor are referenced (by aria-describedby for example) by focusable elements, or are read out in an aria-live region will be missed by a screen reader in Forms Mode.

This Bookmarklet allows you to see what will likely be missed by a screen reader in Forms Mode.

When you run it, text that will be missed by a screen reader in forms mode get a dark red border, and text that's referenced by aria gets a dashed green border.

When you run it again, everything gets cleaned up, and the page gets set back to what it was before.

The Bookmarklet

Drag the Nordburg Forms Mode Simulator (FMS) link to your Bookmark toolbar, or add to a Bookmarks folder.
